## Context

当前 TodoAgent 已实现基础 CRUD 功能和 CLI 交互层。根据 proposal.md，需要增强以下功能：时间维度筛选、优先级管理、视觉状态指示、搜索、统计、清理功能、今日个人助理简报、日计划和长期偏好记忆。

现有技术栈：
- Python 3.x
- `rich` 库用于终端美化
- `prompt_toolkit` 用于命令补全和历史
- SQLite 作为默认持久化存储，JSON 作为兼容后端
- OpenAI-compatible API 用于 LLM 交互

## Goals / Non-Goals

**Goals:**
- 实现时间维度筛选（今日/本周/本月）
- 实现任务优先级管理（高/中/低）
- 实现基于状态的颜色视觉区分
- 实现关键词搜索功能
- 实现任务统计功能
- 实现已完成任务清理功能
- 实现今日个人助理简报和日计划
- 实现用户长期偏好记忆，并让 Agent 在系统提示中读取这些偏好
- 保持与现有 API 的兼容性

**Non-Goals:**
- 不引入 SQLite 以外的新数据库依赖
- 不添加图形界面
- 不支持多用户
- 不实现复杂的权限系统
- 不实现后台常驻提醒服务

## Decisions

### 1. 时间筛选实现方案
- **方案**: 在仓储端口与具体仓储中新增 `get_today()`, `get_this_week()`, `get_this_month()`, `get_upcoming()` 方法
- **理由**: 集中管理时间逻辑，便于维护和测试
- **替代方案**: 在 CLI 层处理时间逻辑 → 拒绝，因为工具层也需要这些方法

### 2. 优先级存储方案
- **方案**: 在 Todo 类中添加 `priority` 字段（默认值: "medium"）
- **支持值**: "high", "medium", "low"
- **理由**: 简单直观，向后兼容（现有任务默认中优先级）

### 3. 视觉区分方案
- **方案**: 在 `todo_cli.py` 中使用 Rich 库的颜色系统
- **颜色映射**:
  - 红色: 高优先级 / 已过期
  - 橙色: 即将到期（2天内）
  - 黄色: 中优先级
  - 绿色: 低优先级 / 正常
  - 灰色: 已完成
- **理由**: 提升用户体验，快速识别任务状态

### 4. 搜索实现方案
- **方案**: 在仓储中添加 `search(keyword)` 方法
- **搜索范围**: 标题和描述
- **匹配方式**: 不区分大小写的子字符串匹配
- **理由**: 简单实用，满足日常搜索需求

### 5. 统计实现方案
- **方案**: 在 `todo_manager.py` 中增强 `get_statistics()` 方法
- **统计指标**: 总数、已完成、未完成、过期、即将到期、完成率
- **理由**: 提供全面的任务管理洞察

### 6. 工具扩展方案
- **方案**: 在 `tool_models.py`, `tool_definitions.py`, `tool_executor.py` 中扩展现有工具并添加新工具
- **扩展工具**: `list_todos`（新增时间范围和优先级参数）
- **新增工具**: `search_todos`, `clear_completed`, `remember_preference`, `list_preferences`, `forget_preference`
- **理由**: 让 AI Agent 能够使用新增功能

### 7. 今日简报与日计划方案
- **方案**: 在 CLI 层新增 `/today` 和 `/plan day`，复用仓储查询结果并用本地排序函数生成任务优先级。
- **排序规则**: 已过期任务优先，其次按 high/medium/low 优先级，再按截止时间和创建时间排序。
- **理由**: 这是本地可验证的个人助理体验，不依赖模型输出即可稳定使用。
- **替代方案**: 完全交给 LLM 生成计划 → 拒绝，因为没有工具结果约束时容易不稳定。

### 8. 长期偏好记忆方案
- **方案**: SQLite 后端新增 `assistant_preferences` 表；JSON 后端写入同目录 sidecar `*.preferences.json`。
- **Agent 集成**: `AgentCore._build_system_prompt()` 注入长期偏好；工具层暴露记住、查看、忘记三个操作。
- **理由**: 不改变 Todo 数据结构，避免污染任务表，同时保留 JSON 兼容后端。
- **替代方案**: 把偏好写入短期会话记忆 → 拒绝，因为短期记忆退出后清空，不能满足个人助手长期使用目标。

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| JSON 文件格式变更导致数据迁移问题 | 在 `Todo.from_dict()` 中添加向后兼容逻辑，缺少 `priority` 字段时默认为 "medium" |
| 时间格式不一致导致解析错误 | 使用灵活的时间解析逻辑，支持多种时间格式 |
| 大量任务时搜索性能下降 | 目前数据量较小，采用简单字符串匹配；未来可考虑索引优化 |
| 颜色输出在某些终端不显示 | 使用 ANSI 转义码 + Rich 库的跨平台支持 |
| 长期偏好写入可能记录过多临时信息 | 工具描述要求仅在用户明确要求记住或长期使用时调用 |
| CLI 计划排序可能不符合用户习惯 | 通过长期偏好为后续个性化排序留下扩展点 |
