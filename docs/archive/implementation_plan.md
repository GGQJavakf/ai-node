# 像 Claude Code 一样的 AI 待办管家工具 - 需求与架构整理

## 1. 目标愿景
将现有的依靠“数字菜单”驱动的 CLI 待办工具，升级为**“以自然语言交互为核心，具备主动感知与工具调用能力的 Agent CLI”**。在体验上媲美 Claude Code，同时作为学习开发 AI Agent 的实战项目。

## 2. 核心功能与交互需求 (UX / UI)

### 2.1 会话式体验 (Conversational REPL)
- **取消固定的数字菜单**：进入系统后直接是一个聊天对话流（如 `🤖 TodoAgent > `），支持历史回顾和方向键操作。
- **Markdown 渲染与语法高亮**：利用 `rich` 库美化终端输出，比如动态渲染表格和代办事项的状态颜色。
- **流式输出 & Thinking 状态**：AI 思考与请求大模型 API 时有动态 Spinner (加载动画)，文本生成具有打字机的流式效果，避免长时间等待。

### 2.2 快捷命令 (Slash Commands)
除了自然语言，还需实现系统级命令作为快捷方式：
- `/list` - 表格化展出所有或今日待办任务
- `/add` - 快捷新增指令
- `/history` - 查看任务流转或会话记录
- `/help` - 查看可用的系统快捷键
- `/exit` 或 `/quit` - 安全退出保存

### 2.3 终端命令行增强
- 使用 `prompt_toolkit` 提供命令输入历史、命令智能提示等，让命令行交互体验无限接近成熟的 CLI 工具。

## 3. 学习目标深入：AI Agent 架构设计

在本项目中，我们将不再写一堆 if-else 进行简单对话拦截，而是借此掌握现代 Agent 架构：

### 3.1 核心驱动：Tool Calling (函数调用引擎)
- **能力暴露**：需使用 JSON Schema 定义 `TodoManager` 各功能（如 `add_todo`, `delete_todo` 等），注册为 AI 工具。
- **意图拆解**：当您输入“下周二有个重要的技术评审会议，会前一天记得提醒我准备 PPT”，Agent 不仅会自己切分时间线，更会自动发起两次平行的 `add_todo` 工具调用。

### 3.2 System Prompt 与 Context
- **本地上下文动态注入**：在每次与 LLM 对话时，动态将当前时间（精准到周几/时区）作为系统上下文喂给 AI，从而让它具备“时间观”。
- **会话状态 (Memory)**：将当前未挂起状态、上一次操作的事项 ID 都放入上下文，让 Agent 听得懂诸如“把刚才那个任务改成高优先级”这种**包含代词指代**的语句。

### 3.3 执行循环 (Agent Execution Loop)
- 控制流搭建：`用户诉求` -> `大模型评估是否需要工具` -> `本地执行函数` -> `执行结果扔回给模型` -> `再思考或直接总结成自然语言返回`。完整走通大语言模型的规划循环(Reasoning Loop)。

## 4. 技术栈选型

- **语言**: Python 3.10+
- **CLI 核心引擎**: `prompt_toolkit` (命令行交互驱动), `rich` (UI 渲染和组件输出)。
- **大模型/Agent方案**: 不使用过于黑盒庞大的LangChain，推荐**手写 OpenAI 标准库的 Function Calling 循环**。这能帮您最透彻地理解 AI Agent “它为什么知道要调代码”的底层逻辑。
- **数据存储**: 延续使用之前写好的 `todo_manager.py` JSON 存储。

## 5. 阶段性演进计划 (Roadmap)

- [ ] **阶段一：建立交互终端基座 (CLI UI 层)**。引入新库组合出一个带有状态栏、能够响应斜杠命令(`/?`)与智能补全的聊天循环壳子。
- [ ] **阶段二：封装 Agent 核心调用对象**。改造底层方法并通过 `pydantic` 生成供 LLM 理解的标准化 Tool Descriptions。
- [ ] **阶段三：实现 Agent 原生推理循环 (Reasoning Loop)**。连接线上模型，跑通自然语言转代码调用的核心动作。
- [ ] **阶段四：UI 融合与流式输出**。在模型跑循环时外挂 Spinner 动画及最终结果的流式返回。
